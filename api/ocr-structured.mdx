---
title: POST /api/ocr-structured
description: Extract structured invoice data using MyBillBook schema with reconciliation
---

The `/api/ocr-structured` endpoint extracts invoice data into a structured JSON format compatible with MyBillBook. It includes automatic reconciliation to verify calculations and detect mismatches.

## Endpoint

```
POST /api/ocr-structured
```

## Request Body

<ParamField path="imageBase64" type="string" required>
  Base64-encoded image data or data URL
  
  Formats:
  - Bare base64: `iVBORw0KGgoAAAANS...`
  - Data URL: `data:image/png;base64,iVBORw0KGgo...`
</ParamField>

<ParamField path="mimeType" type="string" default="image/png">
  MIME type of the image (e.g., `image/png`, `image/jpeg`, `image/webp`)
</ParamField>

<ParamField path="model" type="string">
  Override the default model. Examples: `openai/gpt-4o`, `anthropic/claude-3.5-sonnet`
</ParamField>

## Response Schema

The endpoint returns a structured invoice document with three main sections:

### Voucher Object

<ResponseField name="voucher" type="object">
  Invoice header information and totals

  <Expandable title="properties">
    <ResponseField name="invoice_number" type="string">
      Invoice number (e.g., "INV-2024-001")
    </ResponseField>

    <ResponseField name="invoice_date" type="string">
      Invoice date in dd-mm-yyyy format (e.g., "15-01-2024")
    </ResponseField>

    <ResponseField name="invoice_due_date" type="string">
      Due date in dd-mm-yyyy format
    </ResponseField>

    <ResponseField name="invoice_discount" type="string">
      Voucher-level discount amount (numeric string, e.g., "500.00")
    </ResponseField>

    <ResponseField name="invoice_discount_mode" type="string">
      When discount applies: `"before_tax"`, `"after_tax"`, or `""` (unknown)
    </ResponseField>

    <ResponseField name="round_off" type="string">
      Rounding adjustment (numeric string, typically ±1.00)
    </ResponseField>

    <ResponseField name="total_invoice_amount" type="string">
      Final payable amount including taxes and round-off
    </ResponseField>

    <ResponseField name="additional_charges" type="array">
      Optional voucher-level charges (Freight, Packing, etc.)

      <Expandable title="items">
        <ResponseField name="name" type="string">
          Charge name (e.g., "Freight")
        </ResponseField>

        <ResponseField name="amount" type="string">
          Charge amount (numeric string)
        </ResponseField>

        <ResponseField name="tax_rate" type="string">
          GST rate as percentage (numeric string)
        </ResponseField>

        <ResponseField name="amount_includes_tax" type="boolean">
          Whether amount includes GST (true) or is pre-GST (false)
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="reconciliation" type="object">
      Calculation verification results (added by reconciliation engine)

      <Expandable title="properties">
        <ResponseField name="status" type="string">
          `"matched"` or `"unmatched"`
        </ResponseField>

        <ResponseField name="method" type="object">
          Assumptions used in reconciliation
        </ResponseField>

        <ResponseField name="computed_grand_total" type="string">
          Calculated total from line items
        </ResponseField>

        <ResponseField name="difference" type="string">
          Difference between computed and printed total
        </ResponseField>

        <ResponseField name="items_taxable_total" type="string">
          Total taxable amount from items
        </ResponseField>

        <ResponseField name="items_tax_total" type="string">
          Total GST from items
        </ResponseField>

        <ResponseField name="charges_taxable_total" type="string">
          Total taxable amount from charges
        </ResponseField>

        <ResponseField name="charges_tax_total" type="string">
          Total GST from charges
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Requests

<CodeGroup>

```bash cURL
curl -X POST https://your-domain.com/api/ocr-structured \
  -H "Content-Type: application/json" \
  -d '{
    "imageBase64": "data:image/png;base64,iVBORw0KGgo...",
    "model": "openai/gpt-4o-mini"
  }'
```

```javascript JavaScript/TypeScript
const response = await fetch('/api/ocr-structured', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    imageBase64: imageDataUrl,
    model: 'openai/gpt-4o-mini'
  })
});

const invoice = await response.json();

if (invoice.voucher.reconciliation.status === 'matched') {
  console.log('Invoice calculations verified');
} else {
  console.warn('Reconciliation mismatch:', invoice.voucher.reconciliation.difference);
}
```

```python Python
import requests
import base64

with open('invoice.png', 'rb') as f:
    image_data = base64.b64encode(f.read()).decode()

response = requests.post(
    'https://your-domain.com/api/ocr-structured',
    json={
        'imageBase64': f'data:image/png;base64,{image_data}',
        'model': 'openai/gpt-4o-mini'
    }
)

invoice = response.json()

print(f"Invoice: {invoice['voucher']['invoice_number']}")
print(f"Total: ₹{invoice['voucher']['total_invoice_amount']}")
print(f"Status: {invoice['voucher']['reconciliation']['status']}")
```

</CodeGroup>

## Reconciliation Logic

The endpoint automatically runs reconciliation using `lib/invoice.ts`:

1. **Parse extracted JSON** from model response
2. **Compute line item totals**: `taxable = qty × price × (1 - discount%); tax = taxable × tax_rate; total = taxable + tax`
3. **Apply voucher discount** before or after tax based on `invoice_discount_mode`
4. **Add charges** with their GST
5. **Apply round-off** adjustment
6. **Compare** computed total with `total_invoice_amount`
7. **Set status**: `"matched"` if difference ≤ 0.05, else `"unmatched"`

## Schema Constraints

The system prompt enforces these rules:

- **Numbers**: Strings without commas (e.g., `"1234.50"`)
- **Dates**: dd-mm-yyyy format (converts from other formats)
- **Unknown values**: Empty string `""`
- **Tax-exclusive prices**: `item.price` must be pre-GST; model converts if needed
- **Discount combination**: Multiple discounts combined multiplicatively
- **Round-off**: Typically ±1.00 unless explicitly printed

## Limitations

<Warning>
  This endpoint only supports images. For PDF support, use `/api/ocr-structured-v4`.
</Warning>

- Image input only (no PDF support)
- Simpler schema than v4 (no header discounts, TCS, etc.)
- Less sophisticated reconciliation than v4
- Better suited for basic invoices

## Migration to V4

For India GST invoices with complex scenarios, migrate to `/api/ocr-structured-v4`:

- PDF support with multi-page processing
- Header-level discounts (trade, special)
- TCS (Tax Collected at Source)
- HSN-wise tax tables
- Advanced reconciliation with multiple strategies
- Confidence scoring per field
